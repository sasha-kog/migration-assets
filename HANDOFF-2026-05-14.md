## 3. Pre-Step-2-3 punch list

Two items, both blocking the cleanest possible start of Step 2-3 on this automation.

### Item 1 — Fix Step B.6 else-branch handling (spec + re-sweep)

**The bug:** The current Step B.6 spec checks for `display_value == "Condition Met"` on the predicate's `lineId`. For `else` branches there is no own predicate line — the else fires whenever the corresponding `if` is `Condition Not Met`. Result: B3d_else and B3j_else show as `confirmed_unobserved` even though they're likely the dominant paths (124/136 and 124/136 respectively, by inference from their `_met` siblings).

**Fix (spec patch):** When B.6 encounters a branch whose `predicate` starts with `else` in `branch_catalog.json`, look up the corresponding `if` branch in the same stage and check its `Condition Not Met` count across the same runs. Treat that as the `else` branch's candidate set.

**Effort:** ~10 lines of spec text + one paragraph note. Then re-run B.6 only for the affected stage(s).

This is **orchestrator work**, not a sub-agent task. Do it directly.

### Item 2 — Run the Phase B workspace book preflight

**Goal:** Execute the preflight spec'd in `specs/migration-step-2-3-combined.md` ("Phase B Preflight — Book Capability Check") against the PO digitization workspace, producing `~/Kognitos/to_perform_po_digitization/workspace_book_inventory.json` per the schema.

**This is sub-agent work.** Reasons:
- The orchestrator (this conversation) is already at 40% context; spending it on per-line KLang classification + Apollo Client calls + JSON writes is expensive.
- The preflight is well-spec'd and procedural — exactly the kind of work a focused sub-agent does well.
- Output is self-contained: one JSON file + one short report back.

**What the orchestrator needs to do for Item 2:**
